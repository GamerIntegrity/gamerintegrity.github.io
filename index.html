How scanning works

01
Scan starts

The app runs local checks and shows a live scan log while it works.

02
Results load in GamerIntegrity

The browser stays closed. Results fade into the app first so staff can choose what to view.

03
Choose Redacted or Non-Redacted

Redacted hides private identity fields. Non-Redacted keeps full local paths and identifiers for a trusted local staff check.

04
Export only when needed

Report files are created only when the user chooses to export them.

Detections

GamerIntegrity groups detections in a way that is easier for players and staff to understand.

Projects
Cheat projects / builds

Project folders, build outputs, mapper/injector names, and source-style traces.

Launch
Launch traces

Execution artifacts such as Prefetch and AmCache when Windows makes them available.

Downloads
Browser/download hits

Cheat-related downloads, provider domains, source links, and suspicious archive names.

Startup
Running/startup traces

Running processes, startup entries, services, drivers, and similar launch points.

Reversal
Installed app hits

Reverse-engineering or debugging tools that may matter as context, not automatic proof.

Browser
Browser keyword hits

Search/history terms that may support a timeline when paired with stronger traces.

Files / Folders
File and folder hits

Local names and paths matching cheat, injector, cleaner, or source/build patterns.

External Devices
USB / external devices

Connected removable devices and history that may help explain transfers or missing files.

Hardware
PC hardware

Hardware and system context used to understand the scanned machine.

Redacted vs Non-Redacted

Redacted is the safer sharing view. It masks identity fields, local usernames, computer names, report paths, MAC addresses, serial-like values, and USB serial portions where supported.
Redacted
Safe view

Use this for Discord staff rooms, appeals, tickets, or anywhere private machine details should not be exposed.

Non-Redacted
Full local view

Use this only when trusted staff are allowed to see full paths, identifiers, and machine-specific details.

Fair calls

Do not treat one weak clue as a ban button. Strong cases usually have multiple traces pointing the same way, especially when timing lines up.
01
Start with Overview

Read the level and basis before opening every category.

02
Check the timeline

Look for related activity that happened around the same time.

03
Prioritize stronger traces

Projects, builds, launch artifacts, and downloaded cheat files matter more than weak keyword matches.

04
Keep context

Debuggers, disassemblers, and dev tools can be legitimate. Check the surrounding context.

Known limits

Human call required: GamerIntegrity organizes local traces. It does not replace server rules, appeal process, or a real staff call.
  • Names-only file scanning does not prove file contents.
  • Some Windows artifacts may be disabled, missing, cleaned, or unreadable from a live OS.
  • Browser history can be incomplete or have misleading timestamps.
  • Hardware spoofing cannot be proven from one scan without a trusted baseline.
  • Dual-use tools can be legitimate and should be checked with context.
  • A High or Critical result should start a careful check, not end the conversation by itself.

FAQ

Does GamerIntegrity upload my files?

No. GamerIntegrity is designed to scan locally. Sharing only happens if you export a report and send it somewhere yourself.

Why does it need administrator access?

Some Windows evidence areas require elevated access. If the app is not launched as admin, it shows a Restart as Admin button.

Does the browser open after the scan?

No. Results load inside GamerIntegrity first. Exported files are only created when you choose to export them.

Which view should I share?

Use Redacted for sharing. Use Non-Redacted only for a trusted local check where full paths and identifiers are allowed.

Does a Critical result automatically prove cheating?

No. It means strong local traces were found. Staff should check the categories, timeline, and context before making a decision.

Can legit developer tools be flagged?

Some tools are dual-use. Debuggers, disassemblers, and similar apps should be treated as context unless paired with stronger cheat-specific traces.

Why do I see a SmartScreen warning?

Unsigned or new Windows apps can trigger SmartScreen. Download only from the official repository and verify the release source.

Can a player appeal a result?

That depends on the community or tournament rules. GamerIntegrity gives staff organized data; it does not decide server policy.

Can someone delete traces before scanning?

It is possible for traces to be missing, cleaned, or unavailable. That is why staff should check the full context and not rely on one field.

Where do I report bugs or request features?

Use the official GitHub repository issue tracker so the report can be tracked and fixed in the source.
