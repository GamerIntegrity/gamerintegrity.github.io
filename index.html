What it checks

GamerIntegrity looks for local traces that can help staff understand what happened on a PC.

Launch
Was something run?

Looks for traces of loaders, injectors, mappers, spoofers, and other suspicious tools being opened.

Downloads
What was searched or downloaded?

Checks browser and download clues tied to cheat sites, bypasses, source pages, and tools.

Projects
Source and builds

Checks running apps, services, drivers, scheduled tasks, and startup entries.

Files

Context

Shows extra context that can help explain movement, timing, or the PC being checked.

Areas checked for these cheat types

GamerIntegrity checks local Windows and browser traces that can help show whether cheat tools were searched, downloaded, stored, built, launched, or left behind.

  • Launch traces — signs that loaders, injectors, mappers, spoofers, or cheat tools were opened.
  • Downloads — browser and download records tied to cheat sites, providers, tools, source links, or bypass searches.
  • Startup items — running processes, services, drivers, tasks, and startup entries.
  • Projects — source folders, pasted bases, offsets, SDK dumps, build outputs, and cheat-tool projects.
  • Files / folders — names and paths that match known cheat terms, tools, providers, or suspicious project layouts.
  • Browser history — cheat forums, provider domains, bypass searches, game-specific cheat terms, and download pages.
  • External devices — USB and external device history that may help explain timing or device swapping.
  • Hardware — PC identity context. One scan alone does not prove spoofing, but it helps staff understand the setup.
Fair check note: GamerIntegrity looks for traces. Staff should still review timing, context, and stronger evidence before making a decision.

Fair play

GamerIntegrity is not an instant ban button. It helps show what was found on a PC so staff can make a better call.
  • Old downloads, dev tools, browser history, or test folders can show up in a scan.
  • Some tools are dual-use and need context.
  • Strong results usually have multiple traces pointing the same way.
  • Each community should follow its own rules and appeal process.

FAQ

Does GamerIntegrity upload my files?

No. Scans are local. Sharing only happens if someone exports and sends a report.

Does it need an account?

No. GamerIntegrity is non-gate kept and does not require a login to run a check.

Why does it ask for admin?

Some Windows traces need admin access. The app shows Restart as Admin when needed.

Does it open the browser after every scan?

No. Results load inside GamerIntegrity first. Files are only created when exported.

Which report should I share?

Use Redacted for sharing. Use Non-Redacted only for a trusted local check.

Does a Critical result automatically prove cheating?

No. It means strong traces were found. Staff should check the categories, timeline, and context before making a decision.

Can normal tools be flagged?

Yes. Some developer, debugging, or reverse-engineering tools can be legitimate. Treat those as context unless stronger cheat-related traces are also present.

Why do I see a Windows SmartScreen warning?

New or unsigned community apps can trigger SmartScreen. Download only from the official GitHub release page.

Can someone delete traces before a scan?

Some traces can be missing or cleaned. That is why staff should review the full picture instead of one field.

Where do I report bugs?

Use the official GitHub repo so issues and fixes can be tracked publicly.
